# Daily Briefing

整合 Apple Calendar、Reminders、Mail，輸出今日摘要。重點在**郵件分析**：讀取每封信的內容，判斷重要性，建議行動。

## Flow

1. **行事曆**：`calendar_list_events` 取今天事件
2. **提醒**：`reminders_get_lists` 取清單，對每個有 pending 的清單呼叫 `reminders_list`
3. **郵件分析**（核心步驟）：
   a. `mail_list_messages unread_only:true limit:20` 取所有未讀信
   b. 對每封未讀信呼叫 `mail_read_message` 讀取內容
   c. 判斷重要性（見下方標準）
   d. `mail_list_messages unread_only:false limit:20` 取已讀信（最近）
   e. 分析已讀信，找出可刪除的垃圾（見下方標準）
4. 整合輸出

## 重要性判斷標準

**🔴 需要處理**（需要你回覆或採取行動）：
- 有人直接問你問題或等你回覆
- 截止日期、繳費、報名、申請
- 系統警告、錯誤通知
- 主管、教授、重要聯絡人來信

**🟡 值得注意**（需要知道，但不一定要立刻行動）：
- 通知類（行程確認、審核結果、訂單）
- 有用的資訊但不急

**⚪ 可略過**（不需要行動）：
- 自動通知（npm publish 成功、CI 結果）
- 訂閱電子報、行銷信
- 群組通知、社群平台通知

## 可刪除判斷標準（已讀信）

建議刪除：
- 自動系統信（發布成功、部署通知、帳單已付）
- 行銷 / 促銷信
- 已處理完的通知（已讀且超過 3 天）
- 重複性自動信（每次 push 都會收到的 CI 通知）

## Output Format

```
# 📅 今日簡報 — YYYY-MM-DD（週X）

## 行事曆
- HH:MM – HH:MM　事件名稱（行事曆名）
- 全天　　　　　　事件名稱
（無行程時：今天沒有行程）

## 待辦提醒
- [ ] 提醒名稱（截止：MM/DD）
（無待辦時：沒有未完成的提醒 ✓）

## 📬 未讀郵件（N 封）

### 🔴 需要處理
- **寄件人** — 主旨
  > 一句話摘要說明需要做什麼

### 🟡 值得注意
- **寄件人** — 主旨
  > 一句話摘要

### ⚪ 可略過
- 寄件人 — 主旨（原因：自動通知）

## 🗑️ 建議刪除（已讀）
以下這些信已讀且不需要保留：
- 寄件人 — 主旨（原因：自動系統通知）
- 寄件人 — 主旨（原因：行銷信）
```

## Rules

- 行事曆事件按開始時間排序；全天活動列在最前面
- 提醒只顯示未完成（`show_completed: false`），有截止日期的優先
- 郵件**必須讀取內容**才能分類，不能只看主旨猜測
- 若未讀信超過 15 封，先處理最近 3 天的，超過的標注「還有 N 封較舊的未讀」
- 若任何 MCP 工具失敗，跳過該區塊並標注「（無法取得）」，繼續輸出其他區塊
- 輸出完後問：「要幫你處理哪些信嗎？（回覆、刪除、或標記已讀）」
